    How the model works

    A transparent, reproducible pipeline — not a black box.

    1

    Clean & impute

    Missing BMI values are imputed with the dataset median; unknown smoking status is kept as its own category instead of being discarded.

    2

    Encode & scale

    Numeric features are standardized and categorical features are one-hot encoded inside a single scikit-learn pipeline — so training and serving can never drift apart.

    3

    Model selection

    Logistic Regression, Random Forest, and Gradient Boosting are compared with 5-fold cross-validation on ROC-AUC, with class weights balanced for the rare stroke class.

    4

    Serve

    The winning pipeline is saved as a single artifact and loaded directly by this Flask API — no manual re-encoding, no mismatched features.

This project optimizes for ROC-AUC and recall instead, and applies balanced class weighting, so the model is tuned to actually catch at-risk patients rather than to chase a misleadingly high accuracy number.

    About this project

    StrokeGuard AI is a final year engineering project demonstrating an end-to-end machine learning system: data cleaning, a leak-free preprocessing pipeline, model comparison, and a served web application with real-time inference. The training data is the publicly available Stroke Prediction Dataset on Kaggle.
